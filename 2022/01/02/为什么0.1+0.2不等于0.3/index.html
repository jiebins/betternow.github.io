为什么0.1+0.2不等于0.3？

发表于2022-01-02|更新于2022-01-02
|阅读量:

由于JavaScript使用IEEE 754格式来表示数值类型，这个问题是由这个表示方法引起的，因此我们从IEEE 754格式表示方法出发逐步回答这一问题。

IEEE 754介绍

IEEE 754定义了浮点数在计算机中的表示格式。一个浮点数可以用类似科学记数法的格式进行表示。

符号位S

符号位S决定浮点数是正数（S=0）还是负数（S=1）。

指数位E

指数位E可以为正也可以为负，为了在浮点数运算中不用计算太多的补码（负数用补码表示），实际的指数值会加上一个偏置值Bias再进行存储，单精度数的Bias为127，双精度为1023，因此指数的取值范围为：单精度-126~+127，双精度为-1022~+1023。指数位E决定浮点数的范围。

有效数字位M

有效数字位M是二进制小数，有效数字位M决定浮点数的精度。

JavaScript中浮点数是双精度浮点数，共64位，其中第0到51位（共52位）存储有效数字位，第52到62位存储指数位，第63位存储符号位，存储方式如下图。

十进制浮点数化二进制浮点数

要使用IEEE 754格式表示浮点数，我们需要先将十进制的浮点数化为二进制浮点数。转化的步骤如下

整数部分转化为二进制

十进制整数化二进制整数的方法为：除2倒取余数。用2去除十进制整数，可以得到商和余数，再用2去除商，又能得到一个新的商和余数，如此进行直至商为0，最后将余数从后向前排列就能得到结果。

小数部分转化为二进制

十进制小数化二进制小数的方法为：乘2取整数部分。将十进制小数乘2后取整数部分，剩下的小数部分继续进行乘2运算，直至小数部分为0，最后将整数部分从前向后排列就能得到结果。

合并整数部分与小数部分

将整数部分和小数部分像十进制浮点数一样合并在一块，用小数点分隔。

例子

下面举个例子演示一下以上步骤。对于十进制浮点数10.625，先将整数部分10化为二进制。

  • 10÷2=5……0
  • 5÷2=2……1
  • 2÷2=1……0
  • 1÷2=0……1

所以10的二进制表示为1010。再将小数部分0.625化为二进制。

  • 0.625×2=1.25 （整数部分1，小数部分0.25）
  • 0.25×2=0.5（整数部分0，小数部分0.5）
  • 0.5×2=1（整数部分1，小数部分0）

所以0.625的二进制表示为0.101。将整数部分和小数部分合并，得到10.625的二进制表示为1010.101。

IEEE 754格式转化

IEEE 754的格式化值中有效数字的范围M为1<=M<2，因此IEEE 754格式化值的首位总为1，在存储的时候不需要显式表示出来。根据上述说法，对于双精度十进制数10.625，可以表示为1.010101×2^3。存储时符号位为0，指数位为3+1023=1026，有效数字位为010101后面用0补齐(第一位的1省略)，最终存到计算机的格式如下。

0 10000000010 0101010000000000000000000000000000000000000000000000

0.1+0.2的计算

按照浮点数的运算规则，浮点数运算需进行五个步骤——对阶、尾数计算、结果规格化、舍入和溢出判断。下面按照这五个步骤来计算0.1+0.2。

对阶

先将0.1和0.2化为IEEE 754标准形式再进行对阶。经计算可知0.1和0.2化为二进制时是无限循环小数，0.1为$1.1001(1001)×2^(-4)$，0.2为$1.1001(1001)×2^(-3)$。由于运算的精度有限（假设都是双精度浮点数，64位），需要对小数进行截断和舍入，舍入规则是向偶数舍入，即舍入位如果是偶数就直接舍去不进位，是奇数就进位。舍入后0.1表示为

0 01111111011 1001100110011001100110011001100110011001100110011010

0.2表示为

0 01111111100 1001100110011001100110011001100110011001100110011010

接着进行对阶，对阶的原则是小阶对大阶，即指数小的通过调整有效数字的小数点位置使得指数与指数大的相同，同时还要对指数小的浮点数的有效数字进行舍入。0.1的指数比0.2小1，所以需要对0.1的有效数字右移一位，指数加1。

对阶后的0.1表示为

0 01111111100 1100110011001100110011001100110011001100110011001101。

尾数计算

将0.1和0.2的有效数字部分相加，即

    0.1100110011001100110011001100110011001100110011001101
+
     1.1001100110011001100110011001100110011001100110011010
= 
   10.0110011001100110011001100110011001100110011001100111

结果规格化

可以看到尾数计算的结果已经大于2，需要将小数点左移1位，指数+1，得到结果为

0 01111111101 0011001100110011001100110011001100110011001100110100

溢出判断

最后检查指数位是否溢出，指数位显然没有溢出，因此运算结束。

结果化简

单从二进制浮点数来看并不能看出结果是否为0.3，我们先将结果化为科学计数法的标准形式，即$1.0011….(0011)0100×2^(-2)$，化为十进制，即$2^(-2)+2^(-5)+2^(-6)+2^(-9)+2^(-10)+…+2^(-49)+2^(-50)+2^(-52)=0.30000000000000004440892098500626$，由于JavaScript中浮点数只能精确到17位，所以得到的结果为0.30000000000000004。

总结

本文从IEEE754标准的定义、进制转换、浮点数计算三个方面依次介绍了计算0.1+0.2所用到的方法，并逐步计算出0.1+0.2=0.30000000000000004这一结果。之所以会出现0.1+0.2不等于0.3的情况正是因为浮点数在计算机中存储存在舍入误差，这个误差会在后续的运算中被放大，因此在JavaScript中比较数值大小时尽量不要测试某个特定的浮点数值。
