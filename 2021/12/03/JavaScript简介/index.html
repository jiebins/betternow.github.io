JavaScript简介

发表于2021-12-03|更新于2021-12-03
|阅读量:

JavaScript部分的学习主要参考《Javascript高级程序设计（第四版）》一书。

1 JavaScript概述

Javascript是一种具有函数优先的轻量级、解释型或及时编译型的编程语言。JavaScript由JS解释器逐行进行解释并执行。JavaScript的作用主要有

  • 表单动态校验（JS最初的目的）
  • 网页特效
  • 服务端开发（Node.js）
  • 桌面程序（Electron）
  • APP（Cordova）
  • 控制硬件（Ruff）
  • 游戏开发（cocos2d）

JavaScript由ECMAScript、DOM和BOM组成。

1.1 ECMAScript

ECMAScript是通过ECMA-262标准化的脚本程序设计语言，是JavaScript的一个标准，它定义了这门语言的如下部分

  • 语法
  • 类型
  • 语句
  • 关键字
  • 保留字
  • 操作符
  • 全局对象

ECMAScript版本迭代简述

版本 内容
ES1 与Javascript1.1基本相同，删除了所有浏览器特定的代码
ES2 只做了一些编校工作
ES3 更新字符串处理、错误定义和数值输出，增加对正则表达式、新的控制语句、try/catch异常处理的支持
ES5 厘清ES3存在的歧义，新增JSON对象、方便继承和高级属性定义的方法，以及严格模式
ES6/ES2015 正式支持类、模块、迭代器、生成器、箭头函数、Promise、反射、代理和众多数据类型
ES7/ES2016 只包含少量语法层面的增强
ES8/ES2017 增加异步函数async和await等
ES9/ES2018 增加异步迭代、剩余和扩展属性等
ES10/ES2019 增加数组、字符串等新方法

1.2 DOM

文档对象模型（DOM，Document Object Model）是一个应用编程接口（API），用于在HTML中使用扩展的XML。DOM将整个页面抽象为一组分层节点，通过DOM提供的接口可以对页面中的各种元素进行操作，比如添加、替换、修改、删除。

（这里插一个DOM的图片）

1.3 BOM

浏览器对象模型（Browser Object Model）与DOM类似，也是一组API，用于支持访问和操作浏览器的窗口，通过BOM可以操控浏览器显示页面之外的部分。BOM没有相关标准的JavaScript实现，所以每个浏览器实现的都是自己的BOM。HTML5的出现解决了很多与BOM相关的问题，它以正式规范的形式涵盖了尽可能多的BOM特性。下面是BOM相关的一些扩展

  • 弹出新窗口
  • 移动、缩放和关闭浏览器窗口
  • navigator对象提供关于浏览器的详尽信息
  • location对象提供浏览器加载页面的详尽信息
  • screen对象提供关于用户屏幕分辨率的详尽信息
  • performance对象提供浏览器内存占用、导航行为和时间统计的详尽信息
  • 对cookie的支持
  • 其他自定义对象，比如XMLHttpRequest

2 JavaScript的引入

在HTML文档中引入JavaScript脚本主要有三种方式：行内引入、内嵌引入和外部引入。

2.1 行内引入

行内引入即将单行或少量JavaScript代码写在HTML标签的事件属性中，比如

<button onclick="alert('这是弹框')">点我弹框</button>

行内引入可读性差，引号书写容易出错，在特殊情况下可以使用。

2.2 内嵌引入

内嵌引入需要使用<script></script>标签，先对script标签进行介绍。

2.2.1 script标签

script标签有以下属性

  • async：可选，用于异步加载外部Javascript脚本文件，所以只对外部脚本文件有效。
  • charset：可选，使用src属性指定的代码字符集，很少使用。
  • crossorigin：可选，配置请求的CORS（跨源资源共享）设置。默认不使用CORS。crossorigin="anonymous"表示配置文件请求不必设置凭据标志，crossorigin="use-credentials"设置凭据标志，意味着出站请求会包含凭据。
  • defer：可选，表示脚本可以延迟到文档完全被解析和显示之后再执行，只对外部脚本文件有效。
  • integrity：可选，用于指定加密签名，允许对比接受到的资源和指定的签名以验证子资源完整性，如果签名不匹配页面会报错，脚本也不会执行，用于确保CDN不会提供恶意内容。
  • language：废弃，已被type属性替代
  • src：可选，表示JavaScript外部文件
  • type：可选，表示代码块中脚本语言的内容类型，一般都是"text/javascript"，如果这个值是module，则代码会被当成ES6模块，而且只有这时候代码才能出现import和export关键字。

2.2.2 内嵌引入

直接在<script></script>标签中间放入代码即可完成内嵌引入，比如

<script>
function sayHi() {
    alert("Hi!");
}
</script>

在<script>标签中的代码被计算完成之前，页面的其他内容不会被加载，也不会被显示。同时，使用内嵌引入时要注意代码中不能出现字符串</script>，浏览器解析时会将其当成结束的</script>，比如

<script>
    function sayScript () {
        console.log('</script>');
    }
</script>

浏览器逐行解释行内脚本的方式决定了它在看到字符串时会将其当成结束标签，要避免这个问题只需将/转义即可，即'</script>'改为<\/script>。

2.3 外部引入

外部引入需要使用<script></script>标签和src属性，在src属性中指出外部JavaScript文件的URL，比如

<script src="example.js"></script>

使用外部引入的一些注意点

  • 解释外部JavaScript文件时页面也会阻塞。
  • 在XHTML中可以省略结束标签，在HTML中不能省略。
  • 外部JavaScript文件的扩展名js可以省略。
  • 使用src属性的<script>标签不应该再在标签中再包含其他代码，如果两者都提供浏览器只会执行脚本文件忽略标签内代码。
  • 引用外部域的JavaScript文件会向src指定的路径发送GET请求，这个请求不受浏览器的同源策略限制，但返回并执行的JavaScript则受限制，这个请求仍受页面的HTTP/HTTPS协议的限制。
  • 引用外部域的JavaScript文件时还要确保这个域是可信的，可以使用integrity属性来防范JavaScript文件被恶意修改。
  • 若<script>标签都没有使用defer或async属性，那么浏览器会按照它们在页面中出现的顺序依次解释，后一个标签必须在前一个标签解释完后才能开始解释。

2.4 标签位置

<script>标签可以放在<head>标签中，也可以放在<body>标签中。如果把<script>标签全都放在<head>标签中，就意味着必须把所有JavaScript代码都解释完成后才能开始渲染页面，对于需要很多JavaScript文件的页面来说这会导致渲染明显延迟，在解释期间页面完全空白。因此，应该把所有JavaScript引用都放在<body>标签页面内容的后面，页面会在处理JavaScript代码之前完全渲染页面，即

<!DOCTYPE html>
<html>
    <head>
        <title>title</title>
    </head>
    <body>
        <!--页面内容-->
        <script src="example.js"></script>
    </body>
</html>

2.5 推迟执行和异步执行脚本

前面提到<script>标签有defer和async两个属性，两者的共同点与区别如下

共同点

属性 共同点
defer 只对外部脚本有效
async

不同点

属性 特点
defer 后一个推迟的脚本依然会在前一个推迟的脚本执行之后执行，但实际上也不一定会按顺序执行，因此最好只包含一个defer脚本。
async 后一个脚本可以先于前一个脚本执行，脚本之间必须没有依赖关系，并且async脚本在加载期间不应该修改DOM。

用法

<script defer src="example1.js"></script>
<script async src="example2.js"></script>

在XHTML中使用defer="defer"和async="async"。

2.6 动态加载脚本

可以使用DOM动态新建<script>标签，在<script>标签中使用如下代码

let script = document.createElement('script');
script.src = 'example.js';
document.body.appendChild(script);

其中涉及的DOM新建元素的操作会在后面讲述。

2.7 <noscript>标签

<noscript></noscript>用于给不支持JavaScript的浏览器提供替代内容，当下面两种情况出现时，浏览器将显示<noscript>标签中的内容

  • 浏览器不支持脚本
  • 浏览器对脚本的支持被关闭

除上述两种情况外，浏览器不会渲染<noscript>标签中的内容。
