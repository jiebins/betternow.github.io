JavaScript语言基础（一）——语法与变量声明

发表于2021-12-05|更新于2022-01-14
|阅读量:

大多数主流浏览器目前基本支持ES6，因此接下来关于JavaScript的各项特性都基于ES6展开。

语法

ECMAScript的语法与C语言和Java比较相似，因此如果有这两门语言基础的话会更容易理解ECMAScript的语法。

标识符

标识符是变量、函数、属性或函数参数的名称，组成规则如下

  • 第一个字符必须是一个字母、下划线_或美元符号$。
  • 剩下的其它字符可以是字母、下划线、美元符号或数字。

驼峰命名法

标识符习惯使用驼峰命名法来书写，即第一个单词的首字母小写，后面每个单词首字母大写，比如myCar、doSomethingImportant、firstSecond。

大小写敏感

标识符是大小写敏感的，即myCar和mycar表示的是两个不同的变量或函数。

不能作为标识符的情况

关键字、保留字、true、false和null不能作为标识符。

注释

ECMAScript的注释分为单行注释和块注释，注释语法如下

//单行注释
/*
    块注释
*/

严格模式

ES5中增加了严格模式，用于规范代码。要启用严格模式，需要在脚本开头加上一行

"use strict";

它也可以放在一个函数中，表明这个函数在严格模式下执行

function doSomething() {
    "use strict";
    //函数体
}

严格模式的具体影响会在后面用到它的时候再指出。

语句

ECMAScript的语句以分号结尾，比如

let a = 2;
let c = b + a

末尾的分号不是必需的，但为了防止在压缩代码时删除空行导致出错，所以应该在语句结尾加上分号。

多条语句可以用花括号{}括起来变成代码块，比如

if (flag) {
    flag = false;
    console.log(flag);
}

像if之类的控制语句只有在执行多条语句时要求必须用花括号将语句括起来，如果执行单条语句可以省略花括号，比如

if (flag) console.log(flag);

这种写法容易出错，并且没有代码块的写法清晰，所以还是推荐使用代码块的写法。

关键字与保留字

关键字是指事先定义的、有特殊用途的词语，不能作为标识符或属性名。ES6规定的关键字如下

break        do            in            typeof
case        else        instanceof    var
catch        export        new            void
class        extends        return        while
const        finally        super        with
continue    for            switch        yield
debugger    function    this
default        if            throw
delete        import         try

保留字是将来做关键字用的词语，同样不能用作标识符或属性名。ES6规定的保留字有

// 始终保留
enum
// 严格模式下保留
implements    package        public
interface    protected    static
let            private
// 模块代码中保留
await

变量

ECMAScript的变量是松散类型的，即变量可以保存任何类型的值，每个变量只是一个用于保存任意值的命名占位符。能声明变量的关键字有var、let和const。定义变量的方式为

关键字 变量名

3.1 var关键字

使用var定义一个变量

var msg = 'hello world';

以上例子在定义变量msg的同时还给它赋值'hello world'，即变量初始化。也可以不赋值，即

var msg;

此时变量会保存一个特殊值——undefined。 如果需要定义多个变量可以用逗号分隔每个变量，比如

var msg = 'hello world',
    msg1 = 'hello China',
    msg2 = 'hello Guangzhou';

换行和缩进不是必需的，但可以使代码更清晰，利于阅读。

var声明作用域

使用var声明的变量会成为包含它的函数的局部变量，该变量将在函数退出时被销毁

变量message在函数test中声明，经过函数调用后就被销毁，因此logmsg函数调用时会出错。

若要使以上程序不报错，只需将test函数中变量message前的var去掉即可，此时变量message变成了全局变量，在函数外也能访问到。

全局变量维护难度大，也会造成困惑，因此不推荐使用。

全局作用域与局部作用域的区别

作用域的概念在后面会再详细解释，这里先提及一下。

作用域 特点
全局作用域 直接写在<script>标签中的代码都是全局作用域，全局作用域中的变量在页面打开时创建，页面关闭时销毁，所有全局作用域的变量都是window对象的属性。
局部作用域 又叫函数作用域，函数内部就是局部作用域，局部作用域中的变量只在函数内部起作用，函数执行完毕后变量就被销毁。

同时，let关键字不允许重复声明变量，这样会报错，比如

let a;
let a; // 变量a已声明，会报错

let a;
var a; // 变量a已声明，会报错

var a;

let不会变量提升

let关键字声明的变量不会在作用域中被提升。

可以看到let语句定义的变量在未声明前是不能引用的，在let声明之前的执行瞬间被称为“暂时性死区”，在此阶段引用后面声明的变量会报错。

let的全局声明

let可以直接声明在全局作用域下，但它不会成为window对象的属性。

let的条件声明

使用var声明变量时由于变量会被提升到作用域最前面，然后JavaScript引擎会合并同名变量，所以不用检查变量在前面是否已被声明。但let没有这种机制，所以想要检查let声明的变量在前面是否被声明是不可能的。

for循环中的let和var声明的区别

使用var声明的变量在循环体外面也能访问，但使用let声明的变量只能在循环体内访问，比如

for (var i = 0; i < 5; i++) {}
console.log(i);
for (let j = 0; j < 5; j++) {}
console.log(j); // ReferenceError，j未定义

这个特性在使用定时器时更容易看到区别，比如

for (var i = 0; i < 5; i++) { setTimeout(() => console.log(i), 0) }
// 5 5 5 5 5
for (let j = 0; j < 5; j++) { setTimeout(() => console.log(j), 0) }
// 1 2 3 4 5

第一个for循环中，迭代变量i都是同一个变量，退出循环时，迭代变量i保存的值为5，所以输出的都是5；而在第二个for循环中，每循环一次都声明一个新的迭代变量i，所以每个setTimeout引用的都是不同的变量实例，输出就不相同了。

const关键字

const关键字在声明变量时必须同时初始化变量，给变量赋值，并且在声明后不能再修改变量，比如

const a = 10;
a = 15; // 报错，给常量赋值

const不允许重复声明，并且声明的作用域和let相同，都是块作用域。

总结

本文介绍了ES6的基本语法以及声明变量的三种方式——var、let和const。由于var会造成各种各样不必要的麻烦，因此我们在高版本JavaScript下就不再使用var来声明变量了，改用let和const，并且优先使用const来声明变量，在确定变量在未来会发生改变时才使用let声明，这样会避免因意外赋值导致的非预期行为。
